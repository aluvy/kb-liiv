modal popup

모달팝업 아이디 규칙

  • 버튼 아이디는 btn_화면아이디로 한다.
    ex) btn_MERG06P01
  • 레이어 아이디는 화면아이디로 한다.
    ex) MERG06P01
  • 한 화면아이디에 여러 개의 팝업이 있을경우 _2을 붙인다.
    ex) btn_MERG06P01_2
    ex) MERG06P01_2
  • 모달 팝업을 클릭으로 호출 시, 클릭되는 엘리먼트에는 id가 필수.
  • id는 페이지 내에 유일해야 한다.

레이어팝업 샘플

  • 레이어 팝업에는 우측 상단에 X 버튼이 없다.
  • 팝업 타이틀은 호출 시 title 옵션에서 지정한다.

알럿팝업 샘플

  • 알럿팝업은 레이어팝업과 유사하지만 좌우 여백이 다르다.

바텀팝업

  • 하단에서 올라오는 팝업이다.
  • max-height가 자동으로 계산되어 적용된다. (창높이-헤더높이)

셀렉트박스 바텀팝업

총 23개

자유타이틀 팝업

  • 타이틀 영역을 자유롭게 사용하고 싶을 때는 팝업레이어에 헤더영역(.ly_hd)을 포함하고 원하는 스타일을 준다.

풀페이지 팝업

  • 팝업 호출 시 type : fullpopup 으로 적용한다.

변수 팝업

  • 팝업 호출을 변수에 넣어서 호출하면 해당 팝업을 open(), close() 함수로 제어할 수 있다.

모달팝업 옵션

팝업 호출 문 예제

$(document).on('click', '#btn_MERG06P01', function(){
    $.ohyLayer({
        titleUse: true,
        title: "타이틀명",
        content: '#MERG06P01',
        type: 'bottom',
        dimAct: true,
    });
});
  • titleUse
    true / false (default: true)
    타이틀 사용 여부
  • title
    문자열 (default: "알림")
    컨펌창 타이틀
  • content
    문자열
    호출할 팝업 HTML컨텐츠의 ID (ex) #MERG06P01
  • type
    alert, confirm, conrifm alert, buttom, photo, fullpopup (default: confirm)
    레이어 타입 지정 (타입 별 디자인과 동작이 다르다.)
  • closeUse
    true / false (default: true)
    오른쪽 상단 닫기버튼 노출 (X버튼에는 {content}_dev id가 자동 생성됨)
  • closeAct
    true / false (default: true)
    오른쪽 상단 닫기버튼 동작 유무
  • dimAct
    true / false (default: falst)
    dim 클릭 시 팝업 닫기 동작 사용 유무
  • openAuto
    true / false (default: false)
    레이어를 변수명으로 호출할 때 정의한다
